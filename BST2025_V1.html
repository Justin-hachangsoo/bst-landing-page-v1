3. Technical Architecture

3.1 Solana 기반 아키텍처

BST는 RWA를 지향하며, Solana 네트워크 위에 SPL 토큰으로 발행되어, 낮은 트랜잭션 비용과 빠른 속도를 제공합니다. 주요 컴포넌트는 다음과 같습니다:

  • BST Token Program: SPL 토큰 발행 및 관리
  • Staking Vault Program: 락업별 스테이킹 풀 운영
  • Governance Program: 온체인 투표 및 제안 시스템
  • Yield Strategy Program: 자동 수익 최적화 알고리즘
  • Banking Bridge Program: 전통 금융 연계 인터페이스

3.2 Master LLM 통합 기술

오프체인에서 LLM 모델이 자연어 명령을 처리하고, Solana 프로그램에 안전하게 명령을 전달합니다. “사용자 입력 → LLM 파싱 → 의도 분석 → Solana 트랜잭션 생성 → 프로그램 호출 → 결과 반환”

3.3 Single-Chain Focus

현재 프로토콜은 Solana 네트워크에 집중하여 배포되며, 멀티체인 지원은 후속 로드맵에 포함됩니다.

6. Security & Audit Framework

6.1 External Audit Schedule

프로토콜 보안을 위해 ICO 이후부터 단계적으로 감사 프로그램을 운영합니다.

6.2 Formal Verification

BST는 스테이킹, 보상 분배, 거버넌스 등 핵심 온체인 로직이 명세된 대로 항상 동작함을 보장하기 위해 형식 검증(Formal Verification)을 적용합니다. 총발행량 초과, 보상 계산 오류, 권한 남용과 같은 주요 리스크를 사전에 제거하도록 설계되었습니다.

  • 핵심 불변식(예시): total_supply ≤ MAX_SUPPLY, pool_balance ≥ Σ claimable_reward[u], staked_amount[u]는 언스테이킹 전까지 감소하지 않음, 모든 파라미터 변경은 승인된 거버넌스 절차(제안→투표→타임락→실행)로만 가능, tokenized_rwa ≤ oracle_value × collateral_ratio
  • 검증 절차: (1) 불변식/전·후조건 명세 → (2) Rust(Anchor) 코드 어노테이션 & TLA+ 등 모델링 → (3) SMT 솔버·모델 체커(Kani/Prusti 등) 자동 증명 → (4) 퍼징(cargo-fuzz)·유닛 테스트 병행 → (5) CI 파이프라인(GitHub Actions)으로 PR마다 자동 검증

한계 고지 — 형식 검증은 정의된 명세 범위 내에서만 완전성을 보장합니다. 오라클·오프체인 데이터, 사용자 키 관리 등은 별도의 보안 프로세스를 통해 관리되며, 컨트랙트 변경 시 재검증 결과를 공개 보고서로 투명하게 제공합니다.

6.3 Insurance Integration

BST는 스마트컨트랙트 취약점, 브리지·오라클 실패 등 핵심 온체인 리스크를 완화하기 위해 Nexus Mutual, InsurAce 등 탈중앙 보험 프로토콜과 연계한 커버 모델을 도입합니다. 커버 범위·기간·프리미엄(예: 연 1~4% 수준)은 DAO 의사결정에 따라 설정되며, 컨트랙트 업그레이드 및 주요 파라미터 변경 시 자동으로 커버 상태가 갱신되도록 설계합니다.

  • 커버 범위: 스마트컨트랙트 취약점/해킹, 브리지·오라클 오류, 스테이블코인 디펙 등
  • 연계 프로토콜: Nexus Mutual(Smart Contract Cover 등), InsurAce(Smart Contract Vulnerability / De-peg Cover 등)
  • 프리미엄 & 한도: 예시 기준 연 1~4% 수준(한도·기간·리스크 점수에 따라 변동)
  • 구매/유지 방식: DAO 트레저리로 기본 커버 일괄 구매 + 사용자 선택형 추가 커버 옵션 제공
  • 클레임 절차: 사고 발생 → 온체인 증빙 제출 → 보험 프로토콜 커뮤니티/심사 DAO 승인 → 보상금 분배
  • 투명성: 커버 대상·프리미엄 지출·클레임 결과를 정기 보고서로 공개하고, 거버넌스에 반영

면책 및 한계 — 보험 커버는 명시된 범위 내에서만 적용되며, 개인 지갑 키 유실·피싱 등 오프체인 리스크는 제외됩니다. 각 프로토콜의 정책 변경에 따라 커버 범위가 조정될 수 있으며, 갱신 사항은 DAO를 통해 공지됩니다.

7. Yield Strategies

7.1 Dynamic Interest Rate

시장 상황에 따라 보상률을 자동 조정하는 모델을 사용합니다.

보상률 = 기본 이율 + (총 스테이킹 비율 × 변동계수)

7.2 Auto-Optimization Vaults

BST는 Solana 기반 여러 디파이 프로그램을 연동해 자동 리밸런싱과 복리(재투자) 효과를 극대화하는 전략형 볼트를 제공합니다. 볼트 내 자산은 리퀴드 스테이킹(Liquid Staking) 토큰과 하이브리드 스테이킹(유동성 유지 + 고정 락업 병행) 상품을 조합하여 유동성과 수익률을 동시에 확보합니다. 특히 하이브리드 스테이킹은 멀티체인 스테이킹 방식을 적용하며 베스팅(Vesting)이 없어 즉시 유동성 확보가 가능하므로, 초기·지속 수요를 폭증시킬 것으로 기대됩니다.

  • 전략 엔진: APR, TVL, 슬리피지, 수수료 등 온체인 지표를 실시간 분석해 자산 비중을 자동 조정
  • 복리 구조: 주기적으로 발생한 보상을 자동 재투자하여 수익을 누적 증대
  • 리스크 관리: 오라클 이상/프로토콜 비정상 시 보수적 모드로 전환, 사전 정의된 리스크 한도 준수
  • 거버넌스 연계: 전략 변경·파라미터 수정은 DAO 제안→투표→타임락 절차로만 진행
  • 투명성: 볼트 성과, 수수료, 재투자 내역을 온체인 대시보드로 공개
  • 하이브리드 스테이킹 특징: 멀티체인 참여 가능, 베스팅 없음 → 즉시 유동성 + 높은 참여 유인
